# 配置参考

每个配置项是干什么用的。按设置界面的标签页分组说明。

## 通用

| 配置项 | 作用 |
|-------|------|
| 媒体路径 | 放视频文件的目录 |
| 成功输出目录 | 刮完后把文件复制一份到另一个地方，不填就不复制 |
| 失败输出目录 | 刮失败的文件挪到这个目录 |
| 软链接路径 | 创建软链接的目标目录（不移动原文件） |
| 媒体类型 | 视频文件扩展名，默认 mp4/mkv/avi 等 |
| 字幕类型 | 字幕文件扩展名，默认 srt/ass 等 |
| 跳过小文件 | 小于指定大小的文件不处理 |
| 跳过成功文件 | 已经刮成功的文件跳过不处理 |
| 记录成功文件 | 把成功记录写到文件 |
| 使用软链接 | 不移动原文件，创建软链接 |
| 符号链接定义 | 硬链接相关配置 |

## 清理

| 配置项 | 作用 |
|-------|------|
| 清理指定后缀 | 删除某些后缀的文件 |
| 清理指定文件名 | 删除名字包含指定字符串的文件 |
| 清理包含字符串 | 删除文件名里含指定字符串的文件 |
| 清理小于指定大小 | 删除小于某个值的文件 |
| 忽略指定后缀 | 上述清理不碰这些后缀 |
| 忽略包含字符串 | 上述清理跳过名字含指定字符串的文件 |
| 自动清理 | 刮完后自动执行清理 |

## 刮削

| 配置项 | 作用 |
|-------|------|
| 并发数 | 同时刮多少个文件（建议 3-5）|
| 间歇模式 | 每刮完一批停一会儿，防止被网站封 |
| 刮削模式 | info（信息优先）/ speed（速度优先）/ single（单站快速）|
| 读取模式 | 读取模式的 4 个子选项 |
| 更新模式 | 更新模式的目录/文件命名模板 |
| 文件移动 | 成功后移动/重命名，失败后移动 |
| 有码网站 | 有码影片用哪些网站，排前面的优先 |
| 无码网站 | 同上，针对无码 |
| FC2 网站 | 同上，针对 FC2 |
| 国产网站 | 同上，针对国产 |
| 欧美网站 | 同上，针对欧美 |
| 素人网站 | 同上，针对素人 |

## 字段优先级

每个字段独立的来源网站优先级。

支持的字段：标题、原始标题、简介、演员、海报、缩略图、剧照、预告片、发行日期、片长、评分、标签、导演、系列、制作商、发行商。

还可以按刮削类型（有码/无码/FC2/国产/欧美）分别设置不同的字段优先级。

## 命名

使用 Jinja2 模板语法。支持的变量：

| 变量 | 说明 |
|------|------|
| `{{ number }}` | 番号 |
| `{{ title }}` | 标题 |
| `{{ actor }}` | 第一名演员 |
| `{{ all_actor }}` | 全部演员（用逗号隔开）|
| `{{ studio }}` | 制作商 |
| `{{ publisher }}` | 发行商 |
| `{{ series }}` | 系列 |
| `{{ director }}` | 导演 |
| `{{ year }}` | 年份 |
| `{{ release }}` | 发行日期 |
| `{{ runtime }}` | 片长 |
| `{{ score }}` | 评分 |
| `{{ tag }}` | 第一个标签 |
| `{{ ext }}` | 文件扩展名 |
| `{{ cd }}` | CD 序号 |
| `{{ definition }}` | 分辨率（1080p/4K 等）|
| `{{ codec }}` | 编码格式 |

条件渲染示例：
```
[{% if number %}{{ number }}{% endif %}]{{ title }}.{{ ext }}
```

三种命名目标：
- **文件夹命名**：`[{{ number }}]{{ title }}` 这种
- **文件命名**：`{{ number }}.{{ ext }}`
- **NFO 标题**：Emby 库中显示的标题

## 配置文件在哪

程序启动时会找一个叫 `MDCx.config` 的文件（和 main.py 同目录），这个文件里存的是当前启用的配置文件的绝对路径。配置文件是 JSON 格式（v2），如果发现旧版 INI 格式会自动转 JSON。

默认配置文件在 `resources/config/default_config.json`。

用户数据目录（演员数据库、Amazon ASIN 缓存等）在配置文件同目录的 `userdata/` 下。
