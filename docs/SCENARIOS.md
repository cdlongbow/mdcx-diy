# 使用场景和案例

本文档通过实际使用场景和案例，帮助您更好地了解和使用 MDCx 的各项功能。

## 场景 1：新建媒体库

### 场景描述

您刚刚下载了大量视频文件，需要批量整理并建立完整的媒体库，包含元数据、封面、演员信息等，并导入到 Emby/Jellyfin 媒体服务器。

### 操作步骤

#### 1. 首次配置

1. **启动 MDCx**
   ```bash
   # 从源码运行
   python main.py
   
   # 或运行预编译版本
   MDCx.exe
   ```

2. **配置媒体路径**
   - 打开"设置" → "通用"
   - 设置"媒体路径"为存放视频文件的目录（如 `D:/Videos/`）
   - 设置"成功输出目录"（可选，如 `D:/Videos/Organized/`）

3. **配置刮削源**
   - 进入"设置" → "刮削"
   - 为不同类型配置网站优先级：
     - 有码：`JavBus > JavDB > DMM`
     - 无码：`Kin8`
     - FC2：`FC2`
     - 国产：`HDOUBAN > CNMDB`
   - 设置并发数为 3-5
   - 启用失败自动重试

4. **配置下载选项**
   - 进入"设置" → "下载"
   - 勾选需要下载的文件：
     - ✅ 海报（poster）
     - ✅ 缩略图（thumb）
     - ✅ 背景图（fanart）
     - ✅ NFO 文件
    - 启用人脸裁剪

5. **配置翻译**（可选）
   - 进入"设置" → "翻译"
   - 选择翻译引擎（如 Google 翻译）
   - 勾选需要翻译的字段：
     - ✅ 标题
     - ✅ 剧情简介
     - ✅ 标签
   - 启用简繁转换

6. **配置命名规则**
   - 进入"设置" → "命名"
   - 设置文件夹命名模板：
     ```jinja2
     [{% if number %}{{ number }}{% endif %}][{{ studio }}]{{ title }}
     ```
   - 设置文件命名模板：
     ```jinja2
     [{% if number %}{{ number }}{% endif %}]{{ title }}.{{ ext }}
     ```

7. **配置 Emby/Jellyfin 集成**
   - 进入"设置" → "Emby/Jellyfin"
   - 输入服务器地址（如 `http://localhost:8096`）
   - 输入 API Key
   - 勾选需要同步的功能：
     - ✅ 同步演员信息
     - ✅ 同步演员图片
   - 设置并发数为 3

#### 2. 批量刮削

1. **扫描媒体目录**
   - 返回主界面
   - 点击"扫描"按钮
   - 程序会自动扫描媒体目录，列出所有视频文件

2. **选择要刮削的文件**
   - 查看扫描结果
   - 可全选或手动选择需要刮削的文件
   - 可按番号、标题等筛选

3. **开始刮削**
   - 点击"开始"按钮
   - 程序开始批量刮削
   - 观察实时日志和进度

4. **处理结果**
   - 刮削完成后，查看结果报告
   - 检查失败的文件
   - 可重新刮削失败的文件

#### 3. 导入到媒体服务器

1. **确认文件组织**
   - 检查文件是否按命名规则组织
   - 确认 NFO 文件和图片都已生成

2. **刷新 Emby/Jellyfin**
   - 打开 Emby/Jellyfin 管理界面
   - 刷新媒体库
   - 等待扫描完成

3. **验证结果**
   - 检查电影信息是否完整
   - 确认封面、演员信息等显示正常
   - 测试播放功能

### 预期效果

- 📁 文件按规范命名和组织
- 📄 每个视频都有完整的 NFO 文件
- 🖼️ 高清封面和背景图
- 🎭 完整的演员信息和图片
- 🖥️ Emby/Jellyfin 中显示完整元数据

### 常见问题

**Q: 部分文件刮削失败？**
- A: 检查番号格式是否正确，尝试更换刮削源，查看日志了解详细错误。

**Q: 图片下载失败？**
- A: 检查网络连接，或手动下载。

**Q: 翻译不正确？**
- A: 尝试更换翻译引擎，或手动编辑映射表。

## 场景 2：维护现有媒体库

### 场景描述

您的媒体库已经运行了一段时间，部分视频元数据不完整，需要补充演员信息、更新封面、添加背景图等。

### 操作步骤

#### 1. 扫描现有媒体库

1. **配置扫描路径**
   - 打开 MDCx
   - 进入"设置" → "通用"
   - 设置"媒体路径"为现有媒体库目录

2. **检查演员缺失番号**
   - 点击主界面左上角 "检查演员缺失番号" 按钮
   - 输入演员名称查看本地资源中该演员缺失的番号

#### 2. 补充元数据

1. **识别缺失的元数据**
   - 查看扫描结果
   - 识别哪些文件缺少：
     - NFO 文件
     - 海报
     - 背景图
     - 演员 TMDB ID

2. **配置刮削策略**
   - 进入"设置" → "刮削"
   - 配置字段优先级

3. **批量补充**
   - 选择需要补充的文件
   - 点击"开始"
   - 等待刮削完成

#### 3. 更新演员信息

1. **检查演员数据库**
   - 打开 `userdata/actor_database.xlsx`
   - 查看"TMDB ID"列
   - 找出没有 TMDB ID 的演员

2. **通过刮削自动补全**
   - 确认已配置 TMDB API Key
   - 重新刮削包含这些演员的影片
   - 程序会自动查询缺失的 TMDB ID 并写回 Excel

3. **手动验证**
   - 检查自动写回的 TMDB ID
   - 对匹配错误的演员手动修正数据库

#### 4. 同步到 Emby/Jellyfin

1. **配置同步选项**
   - 进入"设置" → "Emby/Jellyfin"
   - 配置服务器地址和 API Key
   - 设置并发数为 3

2. **使用补全工具**
   - 进入"工具" → "补全 Emby 信息及头像"
   - 选择需要补全的演员
   - 点击"开始"

3. **验证结果**
   - 打开 Emby/Jellyfin
   - 刷新演员信息
   - 确认演员信息和图片已更新

### 预期效果

- 📄 缺失的 NFO 文件已补充
- 🖼️ 缺失的图片已下载
- 🎭 演员信息已补全
- 🖥️ Emby/Jellyfin 数据已更新

### 常见问题

**Q: TMDB 查询触发限流？**
- A: 已内置令牌桶限流器（3.5 req/s），如仍触发，请降低并发数并等待一段时间。

**Q: 演员信息不匹配？**
- A: 手动编辑演员数据库，修正日文名、中文名和别名，然后重新刮削。

**Q: Emby 同步失败？**
- A: 检查 API Key 是否有效，确认网络连接，查看详细错误日志。

## 场景 4：字幕管理

### 场景描述

您有一些视频文件没有字幕，或字幕文件命名不规范，需要自动匹配字幕、重新命名并添加到视频目录。

### 操作步骤

#### 1. 扫描无字幕视频

1. **启动字幕管理工具**
   - 进入"工具" → "字幕管理"
   - 设置视频目录

2. **扫描无字幕视频**
   - 点击"扫描"按钮
   - 程序会自动识别没有字幕的视频文件

#### 2. 匹配字幕文件

1. **配置字幕源目录**
   - 设置字幕文件存放的目录
   - 支持多个字幕源目录

2. **自动匹配**
   - 点击"自动匹配"按钮
   - 程序会根据番号自动匹配字幕文件
   - 支持模糊匹配

3. **查看匹配结果**
   - 检查匹配结果
   - 手动调整错误的匹配

#### 3. 复制和重命名

1. **配置命名规则**
   - 设置字幕文件命名模板：
     ```jinja2
     [{% if number %}{{ number }}{% endif %}]{{ title }}.chs.{{ ext }}
     ```

2. **复制字幕**
   - 选择需要复制的字幕
   - 点击"复制到视频目录"
   - 程序会自动复制并重命名

3. **添加 .chs 后缀**
   - 勾选"添加 .chs 后缀"
   - 程序会自动为中文字幕添加 `.chs` 后缀

#### 4. 更新 NFO

1. **重新刮削**
   - 返回主界面
   - 选择已添加字幕的视频
   - 点击"开始刮削"
   - 勾选"仅更新 NFO"

2. **验证结果**
   - 检查 NFO 文件
   - 确认字幕信息已添加

### 预期效果

- 📄 字幕文件已自动匹配
- 📝 字幕文件已正确命名
- 📁 字幕已复制到视频目录
- 📄 NFO 文件已更新字幕信息

### 高级技巧

**批量添加字幕**
- 支持从多个字幕源目录批量匹配
- 支持字幕语言识别
- 支持多字幕文件管理

**字幕质量筛选**
- 配置字幕质量阈值
- 自动过滤低质量字幕
- 优先选择高质量字幕

### 常见问题

**Q: 字幕匹配失败？**
- A: 检查字幕文件名格式，尝试手动匹配，或调整匹配算法参数。

**Q: 字幕无法显示？**
- A: 确认字幕编码格式，尝试转换为 UTF-8，检查媒体服务器字幕设置。

**Q: NFO 未更新字幕信息？**
- A: 确认已勾选"仅更新 NFO"并重新刮削，或手动编辑 NFO 文件。

## 场景 5：Amazon 封面更新

### 场景描述

您的媒体库中有一些视频封面质量较低，想要从 Amazon 获取高清封面（SL1500）进行替换。

### 操作步骤

#### 1. 识别需要更新的封面

1. **扫描媒体库**
   - 启动 MDCx
   - 扫描媒体目录
   - 查看所有视频的封面

2. **识别低质量封面**
   - 筛选出封面分辨率低于 500x500 的视频
   - 或手动选择需要更新的视频

#### 2. ASIN 识别

1. **自动识别 ASIN**
   - 选择需要更新封面的视频
   - 右键 → "识别 ASIN"
   - 程序会尝试从视频文件中提取 ASIN 条码

2. **查看识别结果**
   - 检查识别的 ASIN 是否正确
   - 手动修正错误的 ASIN

#### 3. 搜索 Amazon

1. **搜索产品**
   - 右键 → "搜索 Amazon"
   - 程序会使用 ASIN 在 Amazon 上搜索产品
   - 显示搜索结果

2. **选择产品**
   - 查看搜索结果
   - 选择最匹配的产品
   - 或手动输入 ASIN

#### 4. 下载高清封面

1. **配置下载选项**
   - 进入"设置" → "Amazon"
   - 设置封面质量为 "SL1500"
   - 启用"自动替换现有封面"

2. **批量下载**
   - 选择所有需要更新封面的视频
   - 右键 → "从 Amazon 下载封面"
   - 等待下载完成

3. **验证结果**
   - 检查下载的封面质量
   - 确认封面已正确替换

#### 5. 更新 NFO

1. **重新刮削**
   - 选择已更新封面的视频
   - 点击"开始刮削"
   - 勾选"仅更新 NFO"

2. **验证 NFO**
   - 检查 NFO 文件
   - 确认封面 URL 已更新

### 预期效果

- 📸 1500x1500 高清封面已下载
- 🖼️ 低质量封面已替换
- 📄 NFO 文件已更新
- 🖥️ 媒体服务器显示高清封面

### 高级技巧

**ASIN 缓存**
- ASIN 查询结果会自动缓存
- 避免重复查询
- 提高查询效率

**智能匹配**
- 启用"软匹配"模式
- 提高匹配准确度
- 支持模糊匹配

### 常见问题

**Q: ASIN 识别失败？**
- A: 手动输入 ASIN，或尝试使用番号搜索 Amazon。

**Q: Amazon 搜索无结果？**
- A: 确认 ASIN 是否正确，尝试搜索番号或标题。

**Q: 封面质量不理想？**
- A: 尝试其他尺寸（如 SL500），或从其他网站下载封面。

## 场景 6：自定义命名规则

### 场景描述

您需要自定义文件和文件夹的命名规则，以满足个人喜好或媒体服务器的特殊要求。

### 操作步骤

#### 1. 了解 Jinja2 模板语法

**基本语法**
```jinja2
# 变量
{{ variable }}

# 条件
{% if condition %}...{% endif %}

# 循环
{% for item in items %}...{% endfor %}

# 过滤器
{{ variable|filter }}
```

**可用变量**
- `number`：番号
- `title`：标题
- `actor`：演员列表
- `studio`：制作商
- `release`：发行日期
- `year`：年份
- 等等...

#### 2. 配置文件夹命名

1. **打开命名设置**
   - 进入"设置" → "命名"
   - 找到"文件夹命名模板"

2. **配置模板**
   ```jinja2
   # 简单模板
   [{{ number }}] {{ title }}

   # 包含制作商
   [{{ number }}][{{ studio }}]{{ title }}

   # 包含演员
   [{{ number }}]{{ title }}-{{ first_actor }}

   # 条件渲染
   [{% if number %}{{ number }}{% endif %}]{{ title }}
   ```

3. **预览效果**
   - 输入测试数据
   - 点击"预览"
   - 查看命名效果

#### 3. 配置文件命名

1. **打开文件命名设置**
   - 找到"文件命名模板"

2. **配置模板**
   ```jinja2
   # 简单模板
   {{ number }}.{{ ext }}

   # 包含标题
   {{ number }}-{{ title }}.{{ ext }}

   # 包含标签
   {{ number }}[{{ tags }}].{{ ext }}

   # 复杂模板
   [{% if number %}{{ number }}{% endif %}][{{ studio }}]{{ title }}[{{ resolution }}].{{ ext }}
   ```

3. **预览效果**
   - 输入测试数据
   - 查看命名效果

#### 4. 配置媒体库命名

1. **打开媒体库命名设置**
   - 找到"媒体库命名模板"

2. **配置模板**
   ```jinja2
   # 简单模板
   {{ title }} ({{ number }})

   # 包含演员
   {{ title }} - {{ all_actor }}

   # 包含年份
   {{ title }} ({{ year }})
   ```

3. **预览效果**
   - 查看媒体服务器显示效果

#### 5. 应用命名规则

1. **批量重命名**
   - 选择需要重命名的文件
   - 右键 → "批量重命名"
   - 等待重命名完成

2. **验证结果**
   - 检查文件和文件夹命名
   - 确认符合预期

### 常用模板示例

**按照番号组织**
```jinja2
# 文件夹
[{{ number }}] {{ title }}

# 文件
{{ number }}.{{ ext }}
```

**按照制作商组织**
```jinja2
# 文件夹
{{ studio }}/{{ number }} {{ title }}

# 文件
{{ number }}.{{ ext }}
```

**按照演员组织**
```jinja2
# 文件夹
{{ first_actor }}/{{ number }} {{ title }}

# 文件
{{ number }}.{{ ext }}
```

**按照年份组织**
```jinja2
# 文件夹
{{ year }}/{{ number }} {{ title }}

# 文件
{{ number }}.{{ ext }}
```

### 高级技巧

**字符清理**
- 自动移除非法字符
- 统一分隔符
- 移除特殊符号

**条件渲染**
- 只在变量存在时显示

## 场景 7：多语言环境配置

### 场景描述

您需要在多语言环境下使用 MDCx，要求元数据显示为中文或日文，并配置翻译和映射表。

### 操作步骤

#### 1. 配置界面语言

1. **打开语言设置**
   - 进入"设置" → "通用"
   - 找到"界面语言"

2. **选择语言**
   - 简体中文
   - 繁体中文
   - English

3. **应用设置**
   - 点击"应用"
   - 重启程序

#### 2. 配置元数据语言

1. **打开字段配置**
   - 进入"设置" → "字段"
   - 为每个字段配置语言

2. **配置标题语言**
   - 选择"标题"字段
   - 设置优先级：日文 > 中文 > 英文
   - 启用翻译

3. **配置演员语言**
   - 选择"演员"字段
   - 设置优先级：中文 > 日文 > 英文
   - 启用翻译

4. **配置其他字段**
   - 根据需求配置其他字段
   - 标签、简介等

#### 3. 配置翻译引擎

1. **打开翻译设置**
   - 进入"设置" → "翻译"

2. **选择翻译引擎**
   - Google 翻译
   - 百度翻译
   - DeepL 翻译
   - LLM 翻译

3. **配置 API Key**
   - 根据选择的翻译引擎配置 API Key
   - DeepL 需要 API Key
   - LLM 需要配置 API 端点和 Key

4. **选择翻译字段**
   - 勾选需要翻译的字段
   - 标题、简介、标签等

5. **配置降级策略**
   - 设置翻译引擎优先级
   - 启用翻译失败时保留原文

#### 4. 配置映射表

1. **打开映射表设置**
   - 进入"设置" → "翻译"
   - 找到"映射表"

   - 点击"编辑演员映射"
   - 添加演员名称映射：
     ```
     松下紗栄子 = 松下纱荣子
     JULIA = 京香JULIA
     ```
   - 保存映射表

3. **编辑制作商映射表**
   - 点击"编辑制作商映射"
   - 添加制作商名称映射：
     ```
     S1 NO.1 STYLE = S1
     PRESTIGE = プレステージ
     ```
   - 保存映射表

4. **编辑标签映射表**
   - 点击"编辑标签映射"
   - 添加标签映射：
     ```
     巨乳 = 大胸
     单体作品 = 个人作品
     ```
   - 保存映射表

#### 5. 测试翻译

1. **刮削测试**
   - 选择一个测试视频
   - 开始刮削
   - 查看翻译结果

2. **检查翻译质量**
   - 检查标题翻译是否准确
   - 检查演员翻译是否正确
   - 检查标签翻译是否符合预期

3. **调整配置**
   - 根据测试结果调整配置
   - 更换翻译引擎
   - 更新映射表

## 场景 8：自动化工作流

### 场景描述

您希望建立自动化的刮削工作流，定期扫描新文件、刮削元数据、同步到媒体服务器，实现完全自动化的媒体库管理。

### 操作步骤

#### 1. 配置自动化刮削

1. **配置监控目录**
   - 进入"设置" → "通用"
   - 设置"媒体路径"为下载目录

2. **配置自动刮削**
   - 启用"下载后自动刮削"
   - 设置刮削延迟（如 5 分钟）
   - 配置刮削选项

#### 2. 配置自动同步

1. **配置 Emby/Jellyfin 自动同步**
   - 进入"设置" → "Emby/Jellyfin"
   - 启用"刮削后自动同步"
   - 配置同步选项

2. **配置演员信息同步**
   - 启用"自动同步演员信息"
   - 启用"自动同步演员图片"
   - 设置同步间隔

#### 3. 创建定时任务

**Windows 任务计划程序**
```cmd
# 创建任务
schtasks /create /tn "MDCx 自动刮削" /tr "C:\MDCx\MDCx.exe --auto" /sc daily /st 02:00

# 手动触发
schtasks /run /tn "MDCx 自动刮削"
```

**Linux Cron**
```bash
# 编辑 crontab
crontab -e

# 添加定时任务（每天凌晨 2 点运行）
0 2 * * * /path/to/mdcx/main.py --auto
```

**macOS Launchd**
```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>Label</key>
    <string>com.mdcx.auto</string>
    <key>ProgramArguments</key>
    <array>
        <string>/path/to/mdcx/main.py</string>
        <string>--auto</string>
    </array>
    <key>StartCalendarInterval</key>
    <dict>
        <key>Hour</key>
        <integer>2</integer>
        <key>Minute</key>
        <integer>0</integer>
    </dict>
</dict>
</plist>
```

#### 4. 配置日志和通知

1. **配置日志**
   - 进入"设置" → "日志"
   - 启用"记录详细日志"
   - 设置日志保存路径
   - 配置日志轮转

2. **配置通知**
   - 启用"刮削完成通知"
   - 配置邮件通知
   - 配置系统通知

#### 5. 监控和调试

1. **查看日志**
   - 定期检查日志文件
   - 查看刮削结果
   - 识别问题

2. **调整配置**
   - 根据日志调整配置
   - 优化刮削策略
   - 提高效率

### 预期效果

- ⏰ 定时自动刮削新文件

- 📝 详细日志记录
