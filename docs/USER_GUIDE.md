# 使用手册

日常使用 MDCx 会遇到的各种操作和问题。

## 界面说明

主界面分成几块：

- **顶部**：工具栏（开始、停止、扫描等按钮）
- **左侧**：站点列表树（显示各个网站的可用状态）
- **中间**：主工作区（刮削结果展示、图片预览、日志）
- **底部**：进度条和状态信息

设置界面通过顶部菜单「设置」进入，分为多个标签页。

## 基本操作流程

### 1. 配置媒体路径

设置 → 通用：
- **媒体路径**：放视频文件的文件夹
- **成功输出目录**：（可选）处理好后复制一份到另一个地方
- **软链接路径**：（可选）创建软链接的目标目录

### 2. 选网站源

设置 → 刮削：
- 按影片类型分别配置：有码、无码、FC2、国产、欧美、素人
- 每个类型可以选多个网站，拖动排序决定优先级
- **免墙通道**（missav_api、r18dev、javdb_api）默认没启用，需要在这手动加到列表

### 3. 字段优先级

设置 → 字段优先级：
- 每个字段（标题、简介、演员、海报等）都可以独立配置来源网站顺序
- 比如标题优先用 JavBus，评分优先用 DMM
- 每种刮削类型（有码/无码等）还可以有不同的字段优先级配置

### 4. 配置下载

设置 → 下载：
- 勾选需要下载的内容：海报、缩略图、背景图、剧照、预告片
- 人脸裁剪：开启后自动检测人脸位置裁剪海报
- Amazon 高清封面：从 Amazon 搜索高清图

### 5. 翻译（可选）

设置 → 翻译：
- 选翻译引擎（Google / Bing 不需要配置，直接用）
- 选需要翻译的字段
- 可选启用简繁转换

### 6. 命名规则

设置 → 命名：
- 用 Jinja2 模板定义文件命名规则
- 常用模板：`[{{ number }}]{{ title }}.{{ ext }}`
- 也可以单独设置文件夹命名和 Emby 标题展示格式

### 7. 开始刮削

1. 回主界面，点「扫描」
2. 勾选要处理的文件（新用户建议先选 1 个试试）
3. 点「开始」，看日志区跑进度
4. 刮完后查结果报告，处理失败的文件

## 四种模式怎么选

| 你的需求 | 选哪个模式 |
|---------|-----------|
| 新下了视频，要完整的海报墙 | 正常模式 |
| 只要按番号归类文件，不要图片 | 整理模式（视频模式） |
| 已有 NFO，要调整目录结构 | 更新模式 |
| 维护已有媒体库，补刮或更新 | 读取模式 |

详细区别看 [FEATURES.md](FEATURES.md) 的「四种刮削模式」。

## 常见问题

### 安装和运行

**装 Python 依赖报错**：
```bash
uv sync --dev
```
如果报某个包找不到，先确认网络能连 PyPI。

**Linux 界面花屏或打不开**：
```bash
sudo apt install libxcb-xinerama0 libxcb-cursor0
export QT_QPA_PLATFORM=xcb
```

**界面太大或太小**：设置 → 界面外观 → 缩放比例，选一个合适的，保存后重启。

### 刮削问题

**刮不出数据**：
1. 检查设置里有没有勾选对应类型的网站
2. 试试设置 → 网络 → 勾选「网络连通性检查」，看网站可不可达
3. 配置代理（设置 → 网络）

**有的网站一直失败**：可能是被墙了。试试：
- 启用免墙通道（在刮削设置里加 missav_api、r18dev、javdb_api）
- 或者开启内置 CF Bypass（设置 → 网络 → 启用内置 Bypass）

**刮出来的标题是日文/英文**：设置 → 翻译，开启翻译并选 Google 或 Bing。

### 演员数据库

**数据库在哪**：`userdata/actor_database.xlsx`

**数据库坏了**：有备份就用备份恢复。可以定期备份这个文件。

**演员 TMDB ID 查不到**：
- 换几种名字搜（日文名、别名）
- 先去 TMDB 网站确认这个演员有没有收录
- 手动在 Excel 里填上 TMDB ID

**TMDB 请求太快报错（429）**：调低并发数（建议 3），开启限流（默认每秒 3.5 次），等一会儿再试。

### 网络

**怎么拿 TMDB API Key**：
1. 打开 https://www.themoviedb.org/
2. 注册账号
3. 在设置里申请 API Key
4. 填到 MDCx 设置里

**配置代理**：设置 → 网络 → 代理，支持 HTTP/HTTPS/SOCKS5。可以指定特定网站走代理。

**NFO 在 Emby 里显示不出来**：
- 确认刮削时勾了「下载 NFO」
- 刷新 Emby 媒体库
- 检查 NFO 文件格式是否正确

## 实际场景

### 场景：新建媒体库

第一次用 MDCx 建立完整媒体库的操作路径：

1. 安装好程序，设置媒体路径
2. 刮削设置里配好有码/无码网站源
3. 下载设置勾选海报、缩略图、NFO，开启人脸裁剪
4. 翻译选 Google，勾标题和简介
5. 选正常模式，扫描 → 勾选全部 → 点开始
6. 等刮完后，在 Emby 添加媒体库指向输出目录，刷新
7. 再到工具里同步演员信息到 Emby

### 场景：日常补新片

每周下载新视频后的操作：

1. 把新视频扔到媒体文件夹
2. 开 MDCx，点扫描
3. 选新增的文件，点开始
4. 检查失败列表，对失败的文件单独处理

### 场景：FC2 管理

FC2 番号格式特殊（纯数字或 FC2-PPV-数字），可以在刮削设置里单独配置 FC2 的网站源。

### 场景：调整命名规则

做完一批刮削想改目录结构：
1. 在设置 → 命名里改好新模板
2. 选读取模式，勾选「有 NFO 时更新」
3. 点开始，程序会按新规则重新整理所有文件
