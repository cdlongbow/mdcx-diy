## 快速开始

### 首次启动

1. 运行程序后，你会看到主界面
2. 程序会自动检测并加载演员数据库（`resources/userdata/actor_database.xlsx`）
3. 首次使用时，建议先检查配置设置

### 界面介绍

主界面包含以下主要区域：
- **菜单栏**：文件、编辑、工具、帮助
- **工具栏**：常用操作按钮
- **数据面板**：显示演员信息和搜索结果
- **状态栏**：显示当前操作状态和提示信息

## 主要功能

### 1. 演员数据库管理

- 查看演员信息（姓名、别名、出生日期等）
- 搜索和筛选演员
- 编辑演员信息
- 导入/导出数据库

### 2. TMDB 数据刮削

- 从 TMDB 获取演员详细信息
- 批量刮削多个演员
- 自动匹配和补全数据
- 限速控制避免 API 超限

### 3. 数据格式化

- 自动格式化数据库
- 修复超链接
- 清理重复数据
- 优化存储结构

## 基本操作

### 搜索演员

1. 在搜索框中输入演员姓名
2. 选择搜索范围（中文名、日文名、别名等）
3. 点击搜索或按 Enter 键
4. 查看搜索结果

### 编辑演员信息

1. 选择要编辑的演员
2. 双击或点击编辑按钮
3. 修改相关信息
4. 保存更改

### 导出数据

1. 选择导出格式（Excel、CSV 等）
2. 点击文件 > 导出
3. 选择保存位置
4. 完成导出

## 演员数据库管理

### 数据库结构

演员数据库包含以下字段：

- **ID**：唯一标识符
- **日文名**：演员日文原名
- **中文名**：中文姓名
- **繁体名**：繁体中文姓名
- **别名**：其他名称
- **出生日期**：出生日期
- **TMDB ID**：TMDB 数据库 ID

### 数据维护

#### 格式化数据库

定期格式化数据库以保持数据一致性：

1. 工具 > 格式化数据库
2. 程序会自动：
   - 清理无效数据
   - 修复超链接
   - 统一格式
3. 查看格式化报告

#### 备份数据

建议定期备份数据库：

1. 文件 > 备份数据库
2. 选择备份位置
3. 程序会自动创建带时间戳的备份文件

## TMDB 数据刮削

### 单个演员刮削

1. 搜索或选择演员
2. 右键 > 从 TMDB 刮削
3. 程序会自动：
   - 查询 TMDB API
   - 获取演员信息
   - 更新数据库
4. 查看结果

### 批量刮削

1. 工具 > 批量刮削
2. 选择刮削选项：
   - 全部演员
   - 未匹配演员
   - 自定义范围
3. 设置并发数和限速（默认 3 并发，3.5 req/s）
4. 开始刮削
5. 查看进度和结果

### TMDB 配置

TMDB API 配置：
- 访问 [TMDB API](https://www.themoviedb.org/settings/api) 获取 API Key
- 在设置中输入 API Key
- 保存配置

**注意**：TMDB API 有速率限制，建议不要超过 4 req/s。

## 高级功能

### API 数据补全

演员信息可以通过多个数据源补全：

1. **TMDB API**：电影数据库信息
2. **Wikidata**：维基百科数据

支持与 Emby 和 Jellyfin 媒体服务器集成：

- 自动补全演员信息
- 获取演员简介
- 更新演员头像

配置方法请参阅 [Emby 集成文档](CODE_WIKI.md#emby-集成)。

## 故障排除

### 常见问题

#### 1. TMDB API 限流错误

**症状**：频繁出现 429 Too Many Requests 错误

**解决方案**：
- 降低并发数（建议 3）
- 启用令牌桶限流（默认 3.5 req/s）
- 等待一段时间后重试

#### 2. 数据库加载失败

**症状**：程序启动时无法加载数据库

**解决方案**：
- 检查文件路径是否正确
- 确认文件未被其他程序占用
- 尝试恢复备份文件

#### 3. 搜索无结果

**症状**：搜索演员时没有结果

**解决方案**：
- 检查搜索词是否正确
- 尝试使用别名或不同语言的名称
- 清空缓存后重试

#### 4. GUI 显示异常（Linux）

**症状**：界面显示不正常或崩溃

**解决方案**：
```bash
# 安装缺失的系统库
sudo apt install libxcb-xinerama0 libxcb-cursor0

# 设置环境变量
export QT_QPA_PLATFORM=xcb
```
