# 代码审查检查清单

审查代码时，逐项核对。

## 功能性检查

### 代码功能

- [ ] 代码实现了想要的功能
- [ ] 边界情况（空值、0、负数等）都处理了
- [ ] 出错时程序有妥善处理
- [ ] 没有明显逻辑错误

### 需求对应

- [ ] 代码符合需求文档的要求
- [ ] 需求文档里的每一条都实现了
- [ ] 没有多余的东西，也没有遗漏该有的东西

---

## 代码质量检查

### 代码风格

- [ ] 遵循项目的代码风格规范
- [ ] 能通过 ruff 检查
- [ ] 变量名和函数名起得清楚
- [ ] 注释写得有用

### 代码复杂度

- [ ] 函数不要太长（不超过 100 行）
- [ ] 嵌套不要太深（不超过 4 层）
- [ ] 圈复杂度不超过 10
- [ ] 没有重复代码

### 可读性

- [ ] 代码结构清楚
- [ ] 没有直接写魔法数字和神秘字符串
- [ ] 变量名有意义，不是 a、b、temp 这种
- [ ] 一个函数只做一件事

---

## 性能检查

### 性能问题

- [ ] 没有明显性能瓶颈
- [ ] 数据库查询已优化
- [ ] 网络请求已优化
- [ ] 内存使用合理

### 资源管理

- [ ] 文件用完正常关闭
- [ ] 网络请求用完正常断开
- [ ] 数据库连接用完正常关闭
- [ ] 没有资源泄漏

---

## 安全检查

### 输入验证

- [ ] 用户输入都经过验证
- [ ] 防了 SQL 注入
- [ ] 防了 XSS（跨站脚本）
- [ ] 防了路径穿越攻击

### 敏感信息

- [ ] API 密钥没直接写在代码里
- [ ] 密码没写到日志里
- [ ] 敏感数据存储时加密
- [ ] 调试信息没有留在代码里

---

## 测试检查

### 单元测试

- [ ] 有对应的单元测试
- [ ] 测试覆盖了主要功能
- [ ] 测试覆盖了边界情况
- [ ] 测试覆盖了异常情况

### 测试质量

- [ ] 测试能独立运行
- [ ] 测试名字清楚
- [ ] 用了合适的 fixture
- [ ] 测试代码好维护

---

## 文档检查

### 代码注释

- [ ] 复杂逻辑写了注释
- [ ] 公开的 API 写了文档字符串
- [ ] 注释准确有用
- [ ] 没有过时未更新的注释

### 项目文档

- [ ] README 跟着更新了
- [ ] API 文档跟着更新了
- [ ] 变更日志写了这次改了什么
- [ ] 相关文档都更新了

---

## 兼容性检查

### 平台兼容

- [ ] Windows 上能跑
- [ ] macOS 上能跑
- [ ] Linux 上能跑

### 版本兼容

- [ ] 兼容指定的 Python 版本
- [ ] 兼容项目用的依赖版本
- [ ] 配置文件兼容旧版本
- [ ] 数据格式兼容旧版本
