# 代码审查检查清单

本文档提供 MDCx 代码审查的检查清单。

## 功能性检查

### 代码功能

- [ ] 代码实现了预期的功能
- [ ] 边界情况得到处理
- [ ] 错误处理完善
- [ ] 没有明显的逻辑错误

### 需求对齐

- [ ] 代码符合需求文档
- [ ] 所有需求点都已实现
- [ ] 没有过度设计或设计不足

---

## 代码质量检查

### 代码风格

- [ ] 遵循项目代码风格规范
- [ ] 使用 ruff 检查通过
- [ ] 变量和函数命名清晰
- [ ] 注释恰当且有用

### 代码复杂度

- [ ] 函数长度合理（<100 行）
- [ ] 嵌套层级合理（<4 层）
- [ ] 圈复杂度合理（<10）
- [ ] 没有重复代码

### 可读性

- [ ] 代码结构清晰
- [ ] 没有魔法数字和字符串
- [ ] 使用有意义的变量名
- [ ] 函数职责单一

---

## 性能检查

### 性能问题

- [ ] 没有明显的性能瓶颈
- [ ] 数据库查询优化
- [ ] 网络请求优化
- [ ] 内存使用合理

### 资源管理

- [ ] 文件句柄正确关闭
- [ ] 网络连接正确关闭
- [ ] 数据库连接正确关闭
- [ ] 没有资源泄漏

---

## 安全检查

### 输入验证

- [ ] 用户输入得到验证
- [ ] SQL 注入防护
- [ ] XSS 防护
- [ ] 路径穿越防护

### 敏感信息

- [ ] API 密钥不硬编码
- [ ] 密码不记录日志
- [ ] 敏感数据加密存储
- [ ] 不输出调试信息

---

## 测试检查

### 单元测试

- [ ] 有对应的单元测试
- [ ] 测试覆盖主要功能
- [ ] 测试覆盖边界情况
- [ ] 测试覆盖异常情况

### 测试质量

- [ ] 测试独立可运行
- [ ] 测试名称清晰
- [ ] 使用适当的 fixture
- [ ] 测试可维护

---

## 文档检查

### 代码注释

- [ ] 复杂逻辑有注释
- [ ] 公共 API 有文档字符串
- [ ] 注释准确且有用
- [ ] 没有过时的注释

### 项目文档

- [ ] README 更新
- [ ] API 文档更新
- [ ] 变更日志更新
- [ ] 相关文档更新

---

## 兼容性检查

### 平台兼容

- [ ] Windows 兼容
- [ ] macOS 兼容
- [ ] Linux 兼容

### 版本兼容

- [ ] Python 版本兼容
- [ ] 依赖版本兼容
- [ ] 配置版本兼容
- [ ] 数据格式兼容
