# 刮削模式详解

MDCx 提供三种刮削模式：正常模式、更新模式和读取模式。每种模式适用于不同的使用场景，具有不同的处理规则。

## 模式概览

### 三种模式

| 模式 | ID | 说明 | 适用场景 |
|------|----|----|----------|
| **正常模式** | 1 | 全新刮削，首次整理 | 新下载的文件，没有 NFO |
| **更新模式** | 3 | 更新现有刮削结果 | 需要更新元数据或重新组织 |
| **读取模式** | 4 | 读取已有 NFO，选择性更新 | 维护已有 NFO 的媒体库 |

### 模式选择

在 MDCx 主界面：
1. 点击"模式"按钮
2. 选择对应模式：
   - 🔄 正常模式
   - 🔄 更新模式
   - 📖 读取模式

## 正常模式

### 模式说明

正常模式用于全新刮削，适用于首次整理的视频文件。

### 处理流程

```
1. 扫描视频文件
   ↓
2. 提取番号
   ↓
3. 刮削元数据
   ↓
4. 下载图片
   ↓
5. 生成 NFO
   ↓
6. 重命名文件
   ↓
7. 移动文件
```

### 特点

- ✅ **完整刮削**：所有字段都会重新刮削
- ✅ **下载图片**：下载所有配置的图片
- ✅ **生成 NFO**：创建新的 NFO 文件
- ✅ **文件移动**：按规则移动文件到目标目录
- ✅ **文件重命名**：按命名规则重命名文件

### 配置选项

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `success_file_move` | 成功后移动文件 | `True` |
| `failed_file_move` | 失败后移动文件 | `True` |
| `success_file_rename` | 成功后重命名文件 | `True` |
| `soft_link` | 使用软链接 | `0` (不使用) |

### 适用场景

🎬 **新文件整理**
- 刚下载的视频文件
- 没有任何元数据的文件
- 需要完全重新整理的文件

### 示例

```
场景：下载了一批新视频

1. 选择"正常模式"
2. 设置媒体路径为下载目录
3. 点击"开始刮削"
4. 系统会：
   - 扫描所有视频文件
   - 刮削元数据
   - 下载图片
   - 生成 NFO
   - 按命名规则组织文件
```

## 更新模式

### 模式说明

更新模式用于更新现有刮削结果，适用于需要更新元数据或重新组织的媒体库。

### 处理流程

```
1. 扫描视频文件
   ↓
2. 检查 NFO 文件
   ↓
3. 根据更新模式规则处理
   ↓
4. 更新元数据（可选）
   ↓
5. 下载图片（可选）
   ↓
6. 更新 NFO（可选）
   ↓
7. 按更新模式规则组织文件
```

### 更新模式规则

更新模式使用 4 个配置项定义文件组织规则：

| 配置项 | 说明 | 示例 | 默认值 |
|--------|------|------|--------|
| `update_a_folder` | 更新模式 - A 目录 | `{{ actor }}` | `{{ actor }}` |
| `update_b_folder` | 更新模式 - B 目录 | `{{ number }} {{ actor }}` | `{{ number }} {{ actor }}` |
| `update_c_filetemplate` | 更新模式 - C 文件命名 | `{{ number }}` | `{{ number }}` |
| `update_d_folder` | 更新模式 - D 目录 | `{{ number }} {{ actor }}` | `{{ number }} {{ actor }}` |

### 更新模式逻辑

```python
# A 目录
if main_mode == 3:  # 更新模式
    folder = "{{ actor }}"
    # 或使用 update_a_folder 配置

# B 目录
elif main_mode == 3:
    folder = "{{ number }} {{ actor }}"
    # 或使用 update_b_folder 配置

# C 文件名
file_name = "{{ number }}"
# 或使用 update_c_filetemplate 配置

# D 目录
elif main_mode == 3:
    folder = "{{ number }} {{ actor }}"
    # 或使用 update_d_folder 配置
```

### 特点

- ✅ **保留已有信息**：默认不覆盖已有 NFO
- ✅ **选择性更新**：可配置更新哪些内容
- ✅ **重新组织**：按更新模式规则重新组织文件
- ⚠️ **文件移动规则**：失败时不移动文件
- ⚠️ **命名规则**：使用更新模式的命名规则

### 适用场景

🔄 **更新元数据**
- 需要更新某个字段的值
- 需要补充缺失的信息
- 需要修正错误的信息

📁 **重新组织**
- 需要调整文件组织结构
- 需要更改命名规则
- 需要重新分类整理

### 示例

```
场景：媒体库需要重新组织

当前结构：
/videos/ABP-123/ABP-123.mp4

目标结构：
/videos/松下紗栄子/ABP-123/ABP-123.mp4

步骤：
1. 选择"更新模式"
2. 配置更新模式规则：
   - update_a_folder: {{ actor }}
   - update_b_folder: {{ number }} {{ actor }}
3. 点击"开始刮削"
4. 系统会按新规则重新组织文件
```

## 读取模式

### 模式说明

读取模式用于读取已有 NFO，选择性更新。适用于维护已有完善 NFO 的媒体库。

### 读取模式选项

读取模式包含 4 个选项：

| 选项 | 枚举值 | 说明 |
|------|--------|------|
| **有 NFO 时更新** | `HAS_NFO_UPDATE` | 本地已刮削成功的文件，重新整理分类（按更新模式规则） |
| **无 NFO 时刮削** | `NO_NFO_SCRAPE` | 本地之前刮削失败的文件和没有 NFO 的文件，重新刮削（按更新模式规则） |
| **重新下载** | `READ_DOWNLOAD_AGAIN` | 重新下载图片和资源 |
| **更新 NFO** | `READ_UPDATE_NFO` | 更新 NFO 文件内容 |

### 配置方式

在 MDCx 主界面：
1. 选择"读取模式"
2. 勾选需要的选项：
   - ☑️ 本地已刮削成功的文件，重新整理分类（按更新模式规则）
   - ☑️ 本地之前刮削失败的文件和没有 NFO 的文件，重新刮削（按更新模式规则）
3. 选择是否：
   - ☑️ 重新下载
   - ☑️ 更新 NFO

### 处理流程

```
1. 扫描视频文件
   ↓
2. 检查 NFO 文件是否存在
   ↓
3. 根据读取模式选项处理
   ↓
4. 有 NFO：
   - 选择性更新
   - 重新下载（可选）
   - 更新 NFO（可选）
   ↓
5. 无 NFO：
   - 完整刮削
   - 下载图片
   - 生成 NFO
   ↓
6. 按更新模式规则组织文件
```

### 特点

- ✅ **智能处理**：根据 NFO 存在情况分别处理
- ✅ **选择性更新**：可控制更新哪些内容
- ✅ **保留已有信息**：默认保留已有 NFO 内容
- ⚠️ **流程同正常模式**：但命名按照更新模式规则执行
- ⚠️ **适合二次刮削**：不破坏已有完善信息

### 适用场景

📖 **维护媒体库**
- 需要重新组织文件结构
- 需要更新部分元数据
- 需要补充缺失的图片

🔄 **选择性更新**
- 只更新某个字段
- 只下载某些图片
- 只调整文件组织

### 示例

```
场景：媒体库文件需要重新组织，但保留已有 NFO

当前状态：
/videos/ABP-123/
  - ABP-123.mp4
  - ABP-123.nfo (完善的 NFO)

操作：
1. 选择"读取模式"
2. 勾选"有 NFO 时更新"
3. 不勾选"更新 NFO"（保留已有 NFO）
4. 配置更新模式规则
5. 点击"开始刮削"
6. 系统会：
   - 读取已有 NFO
   - 按更新模式规则重新组织文件
   - 不修改 NFO 内容
```

## 配置说明

### 配置文件位置

配置存储在 `config.json` 中。

### 主模式配置

```json
{
  "main_mode": 1,
  "read_mode": [],
  "update_mode": "c"
}
```

**字段说明**

| 字段 | 类型 | 说明 | 可选值 |
|------|------|------|--------|
| `main_mode` | int | 主模式 | `1`(正常), `3`(更新), `4`(读取) |
| `read_mode` | list | 读取模式选项 | `["has_nfo_update", "no_nfo_scrape"]` |
| `update_mode` | str | 更新模式规则 | `a`, `b`, `c`, `d` |

### 更新模式配置

```json
{
  "update_a_folder": "{{ actor }}",
  "update_b_folder": "{{ number }} {{ actor }}",
  "update_c_filetemplate": "{{ number }}",
  "update_d_folder": "{{ number }} {{ actor }}",
  "update_titletemplate": "[{% if number %}{{ number }}{% endif %}]{% if title and title != number %}{{ title }}{% endif %}"
}
```

### 文件移动配置

```json
{
  "success_file_move": true,
  "failed_file_move": true,
  "success_file_rename": true,
  "soft_link": 0
}
```

**字段说明**

| 字段 | 类型 | 说明 | 默认值 |
|------|------|------|--------|
| `success_file_move` | bool | 成功后移动文件 | `true` |
| `failed_file_move` | bool | 失败后移动文件 | `true` |
| `success_file_rename` | bool | 成功后重命名文件 | `true` |
| `soft_link` | int | 使用软链接 | `0` |

### 配置界面

在 MDCx 主界面：
1. 点击"设置"
2. 进入"刮削"标签
3. 配置相关选项

## 使用场景

### 场景 1：新下载文件（正常模式）

**需求**
- 下载了一批新视频
- 需要完整刮削元数据
- 需要按规则组织文件

**操作**
1. 选择"正常模式"
2. 设置媒体路径
3. 配置命名规则
4. 点击"开始刮削"

**结果**
- ✅ 完整刮削所有元数据
- ✅ 下载所有图片
- ✅ 生成 NFO 文件
- ✅ 按规则组织文件

### 场景 2：更新元数据（更新模式）

**需求**
- 媒体库已有元数据
- 需要更新某个字段
- 需要重新组织文件

**操作**
1. 选择"更新模式"
2. 配置更新模式规则
3. 选择需要更新的字段
4. 点击"开始刮削"

**结果**
- ✅ 按更新模式规则重新组织
- ✅ 选择性更新元数据
- ⚠️ 失败的文件不移动

### 场景 3：维护媒体库（读取模式）

**需求**
- 媒体库已有完善 NFO
- 需要重新组织文件结构
- 需要保留已有 NFO

**操作**
1. 选择"读取模式"
2. 勾选"有 NFO 时更新"
3. 不勾选"更新 NFO"
4. 配置更新模式规则
5. 点击"开始刮削"

**结果**
- ✅ 保留已有 NFO
- ✅ 按更新模式规则重新组织
- ✅ 不破坏已有信息

### 场景 4：补充缺失内容（读取模式）

**需求**
- 部分文件缺少 NFO 或图片
- 需要补充缺失内容
- 保留已有完善内容

**操作**
1. 选择"读取模式"
2. 勾选"无 NFO 时刮削"
3. 勾选"重新下载"
4. 点击"开始刮削"

**结果**
- ✅ 有 NFO 的文件保留
- ✅ 无 NFO 的文件完整刮削
- ✅ 重新下载缺失图片

## 最佳实践

### 模式选择建议

**首次整理**
- 🎬 使用**正常模式**
- 完整刮削所有内容
- 建立规范的组织结构

**日常更新**
- 🔄 使用**更新模式**
- 选择性更新需要的内容
- 避免破坏已有信息

**维护整理**
- 📖 使用**读取模式**
- 智能处理已有内容
- 保留完善信息

### 配置建议

**正常模式**
```json
{
  "success_file_move": true,
  "failed_file_move": true,
  "success_file_rename": true
}
```

**更新模式**
```json
{
  "success_file_move": true,
  "failed_file_move": false,  // 失败时不移动
  "success_file_rename": true
}
```

**读取模式**
```json
{
  "read_mode": ["has_nfo_update", "no_nfo_scrape"],
  "success_file_move": true,
  "failed_file_move": false
}
```

### 注意事项

⚠️ **备份重要**
- 在使用更新模式或读取模式前
- 建议备份现有媒体库
- 避免意外数据丢失

⚠️ **测试验证**
- 先在小范围测试
- 确认结果符合预期
- 再应用到整个媒体库

⚠️ **模式切换**
- 切换模式前确认配置
- 不同模式的命名规则不同
- 可能导致文件位置变化

⚠️ **并发控制**
- 大量文件处理时
- 适当降低并发数
- 避免系统资源不足
